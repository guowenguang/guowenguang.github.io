Spring

一、spring的优点？

1.1：spring是一个免费开源的框架，容器。

1.2：spring是一个轻量级框架，非入侵的。

1、Servlet的生命周期

1.从服务器开启的到服务器结束

2.从第一次调用到服务器关闭

注意：init方法是对Servlet的一个方法，会在Servlet方法第一个加载进行存储执行，destroy是在服务器销毁时执行，也就服务器关闭的时候。

2、service方法和doGet方法和doPost方法

serivce 方法：

​ 可以处理doGet和doPost方法，但是有service方法会优先执行service方法再执行deGet和doPost方法

doGet方法：

​ 处理get方法

doPost方法：

​ 处理Post方法

注意：当service方法中覆写了super.service（req,resq）会先执行serivce方法在来执行doGet方法和doPost方法，如果doGet方法和doPost方法没有就会报405，但serivce方法还是会执行，因此，在serivce方法中最好不好写super.service(req,resq),写了就要写doGet和doPost。

数据流转过程：

​ 浏览器——-》服务器——-》数据库（发送请求过程）

​ 数据库——-》服务器——–》浏览器（响应处理结果）

请求转发学习：（只发送一次请求给服务器）

​ 作用：实现多个servlet联动操作来处理请求，这样避免代码冗余，让servlet的职责更加明确。

使用：

​ req.getRequestDispatcher(“要转发的地址”).forward(req,resp);

​ 转发的地址是servlet的别名

特点：

​ 一次请求，浏览器地址栏信息不改变。

注意：

​ 请求转发后的直接return结束就可以了。

reuqet作用域

​ 解决一次请求内的servlet的数据共享问题

重定向（向服务器发送两次请求）

​ 解决了表单重复提交的问题，以及当前servlet无法请求的问题。

​ 使用：

​ resp.sendRedirect(String uri);

​ 特点：

​ 两次请求，两个request对象。

​ 浏览器地址栏信息改变

时机：

​ 如果请求中有表单数据，而数据又比较重要，不能重复提交，建议使用重定向。

​ 如果请求被servlet接收后，无法进行处理，建议使用重定向定位到处理的资源里面。

2、Session的性质

session是会话的意思

a.浏览网站：开始—–关闭

b.购物、浏览、付款

开始到结束

3、session机制

​ 客户端第一次请求服务器时，服务器会自动产生一个session对象（用于保存客户的信息），

​ 并且每个session对象都会有一个唯一的SessionID（用于区分其他session）；

​ 服务端会产生一个cookie，并且该cookie的name=JSESSIONID，value=服务器sessionID的值；

​ 然后 服务器会在相应客户端的同时 将该cookie发送给客户端，至此客户端就有cookie（JSESSIONId）

​ 因此客户端的cookie就可以和服务端session一一对应（JSEESION —-session）

session：

​ a.session存储在服务端

​ b.session 是在同一个用户（客户）请求时 共享数据

​ c. session机制：第一次客户请求时产生一个sessionID 并赋值给cookie的 jsessionid，然后发送给客户端

4、session的方法

​ String getId() 获取sessionId

​ setAttribute() //是将username保存在session中

​

​ void setMaxInanctiveInterval(秒)：设置最大有效 非活动时间

四种范围对象（小—–》大）

1.pageContext JSP页面容器

2.request 请求对象

3.response 响应对象

4.session 会话对象

以上有四个对象共有的方法：

​ Object getAttribute(String name) : 根据属性名，或者属性值

​ void setAttribute(String name,Object obj ) :设置属性值（新增，修改）

​ void removeAttribute(String name) : 根据属性名，删除对象

a.

pageContext 当前页面有效 （页面跳转后无效）

b.

request 同一次请求有效，其他请求无效（请求转发后有效，重定向无效，因为他要请求两次）

c.

session 同一次会话有效 （无论怎么跳转，都有效，关闭/切换浏览器后无效 ；从登陆到退出）

d.

application 全局变量：整个浏览器都有效，其他浏览器也有效；；；；；关闭服务器和在其他项目中无效

  1. 以上的4个范围对象，通过setAttribute（）赋值 ，通过getAttribute() 取值
  2. 以上范围对象，尽量使用最小的范围。因为对象的范围越大，造成的性能损耗越高

3.1、面向对象有三大特性:封装、继承、多态。

封装( private )：将类的成员变量和成员方法都隐藏起来、不让外部直接访问；
将类内部与外部交互的方法暴露出来，让方法来控制外部类中的成员变量和实现细节进行安全的访问和操作。
继承（extends）：当子类继承父类要用extends关键字来继承父类的方法和属性，当父类的属性和方法带有private关键字的时候，子类不能继承。而父类不能继承子类的方法和属性
多态：多态是同一个行为具有多个不同表现形式或形态的能力。多态就是同一个接口，使用不同的实例而执行不同操作
